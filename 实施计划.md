## 分支一：mv-architecture

**对应报告：第1章 开发环境搭建及MV架构实现**

### 架构

```
Browser → HTTP → SpringBoot
                   ├── Controller（处理请求 + 返回页面）
                   ├── Model/Entity
                   └── Mapper（直接查数据库）

没有 Service 层，没有前端工程，Thymeleaf 服务端渲染
```

### 实现范围

只做一个模块链路示范：**商品浏览 + 简单登录**

| 功能 | 说明 |
|------|------|
| 商品列表页 | Thymeleaf 模板渲染 |
| 商品详情页 | 展示 SKU、图片 |
| 登录/注册 | 表单提交，Session 存登录态 |

### 技术栈

SpringBoot + MyBatis Plus + Thymeleaf + MySQL（单库 ecommerce）

### 项目结构

```
mv-architecture/
├── pom.xml
├── src/main/java/com/ecommerce/
│   ├── controller/
│   │   ├── ProductController.java
│   │   └── UserController.java
│   ├── mapper/
│   │   ├── ProductMapper.java
│   │   └── UserMapper.java
│   └── entity/
│       ├── Product.java
│       ├── Sku.java
│       └── User.java
├── src/main/resources/
│   ├── templates/
│   │   ├── product/
│   │   │   ├── list.html
│   │   │   └── detail.html
│   │   └── user/
│   │       ├── login.html
│   │       └── register.html
│   └── application.yml
└── init.sql
```

### 特点

- Controller 直接调 Mapper，无 Service 层
- 返回 ModelAndView，不返回 JSON
- 页面和后端耦合在同一工程
- 单库单表

## 分支二：mvc-architecture

**对应报告：第2章 MVC架构设计与实现**

### 架构

```
Browser → HTTP → SpringBoot
                   ├── Controller  ← 参数校验，返回 JSON
                   ├── Service     ← 业务逻辑（新增）
                   └── Mapper      ← 数据访问
```

### 相比 MV 的改进

| | MV | MVC |
|---|----|-----|
| 分层 | 2层 | 3层（+Service） |
| Controller | 什么都干 | 只做参数+路由 |
| 业务逻辑 | 散落 Controller | 集中在 Service |
| 返回格式 | ModelAndView | 统一 R\<T\> |
| 异常处理 | 默认白页 | @RestControllerAdvice |
| 认证 | 无 | JWT + 拦截器 |

### 实现范围

以**注册+登录**为主线，把 MVC 三层讲透。

同时还实现：商品浏览（补充 Service 层）、收货地址 CRUD

### 技术栈

SpringBoot + MyBatis Plus + Thymeleaf + MySQL + JWT

### 项目结构

```
mvc-architecture/
├── pom.xml
├── src/main/java/com/ecommerce/
│   ├── controller/
│   │   ├── UserController.java
│   │   ├── ProductController.java
│   │   └── AddressController.java
│   ├── service/
│   │   ├── UserService.java
│   │   ├── impl/UserServiceImpl.java
│   │   ├── ProductService.java
│   │   └── impl/ProductServiceImpl.java
│   ├── mapper/
│   ├── entity/
│   └── common/
│       ├── result/R.java
│       ├── exception/GlobalExceptionHandler.java
│       └── config/JwtInterceptor.java
├── src/main/resources/
│   ├── templates/
│   └── application.yml
└── init.sql
```

### 特点

- Controller → Service → Mapper 清晰三层
- 统一 `R<T>` 返回 JSON
- JWT 认证 + 拦截器
- 全局异常捕获

## 分支三：vue-springboot

**对应报告：第3-7章**

### 架构

```
[Vue3 SPA] ← REST/JSON → [SpringBoot API]
    │                          │
    ├── Pinia                   ├── Controller (纯JSON)
    ├── Vue Router              ├── Service
    ├── Axios                   └── Mapper
    └── Element Plus

去掉 Thymeleaf，前后端彻底分离
后端三模块（user/product/order），连三个数据库
```

### 相比 MVC 的改进

| | MVC | Vue+SpringBoot |
|---|-----|---------------|
| 页面渲染 | 服务端 Thymeleaf | 前端 SPA |
| 前后端 | 同一工程 | 独立工程独立端口 |
| 用户体验 | 整页刷新 | 局部更新 |
| 接口 | JSON+页面混用 | 纯 REST API |
| 前端工程化 | 无 | Vite + TS + 组件化 |
| 数据库 | 单库 | 三库（为微服务准备） |

### 实现范围（全功能）

| 模块 | 功能 |
|------|------|
| 用户 | 注册、登录(JWT)、地址管理、RBAC角色权限 |
| 商品 | 类目树、SPU+SKU+图片、上下架、审核、搜索、Redis缓存 |
| 订单 | 购物车(Redis)、下单(库存预占)、支付(模拟)、订单状态流 |
| 售后 | 退款/退货申请+审核 |
| 营销 | 优惠券模板+领取+下单使用+满减 |
| 商家端 | 商品管理、订单处理、发货、数据看板(ECharts) |
| 平台端 | 用户/商家管理、商品审核、权限管理 |

### 技术栈

| 层 | 技术 |
|---|------|
| 后端 | SpringBoot 3 + MyBatis Plus 3.5 + MySQL 8.0(三库) + Redis 7 + Redisson + JWT |
| 前端 | Vue3 + TypeScript + Vite + Element Plus + Pinia + Axios + ECharts |
| 环境 | Docker(MySQL, Redis) |

## 分支四：microservices

**对应报告：第5章补充（高并发/微服务）**

### 架构变化

vue-springboot 的三个 Service 模块拆成独立进程：
- 引入 Nacos（注册+配置中心）
- 引入 Gateway（统一入口+全局鉴权）
- 引入 Sentinel（限流+熔断+降级）
- 引入 RocketMQ（异步下单削峰+延迟消息超时取消）
- 秒杀模块（Redis Lua + 防超卖 + JMeter 压测）

### 新增内容

| 模块 | 功能 |
|------|------|
| Nacos | 三服务注册发现、配置迁移 |
| Gateway | 路由转发、跨域、JWT 全局鉴权 |
| Sentinel | 网关限流、秒杀接口保护、熔断降级 |
| RocketMQ | 下单异步化、订单超时取消(延迟消息) |
| 秒杀 | 商品预热Redis、Lua原子扣库存、防重复抢购、前端倒计时+验证码 |

### 时间：3天

## 数据库设计（三库）

### db_user

| 表 | 说明 |
|---|------|
| t_user | 用户(id, phone, password, nickname, avatar, status) |
| t_user_address | 收货地址(id, user_id, receiver, phone, province, city, district, detail, is_default) |
| t_merchant | 商家(id, user_id, shop_name, logo, status) |
| t_admin_user | 后台管理员(id, username, password, status) |
| t_role | 角色(id, role_name, role_code) |
| t_permission | 权限(id, perm_name, perm_code, parent_id, type) |
| t_role_permission | 角色-权限 |
| t_user_role | 用户-角色 |

### db_product

| 表 | 说明 |
|---|------|
| t_category | 类目(id, name, parent_id, level, sort) |
| t_product | SPU(id, merchant_id, category_id, name, description, brand, status) |
| t_sku | SKU(id, product_id, spec_info:JSON, price, original_price, image) |
| t_sku_stock | 库存(sku_id, stock, lock_stock, version) |
| t_product_image | 图片(id, product_id, url, sort, type) |
| t_review | 评价(id, user_id, product_id, order_id, rating, content) |

### db_order

| 表 | 说明 |
|---|------|
| t_order | 订单(id, order_no, user_id, total_amount, pay_amount, status) |
| t_order_item | 订单明细(id, order_id, sku_id, product_name, price, quantity) |
| t_payment | 支付(id, order_id, pay_no, amount, pay_method, status) |
| t_logistics | 物流(id, order_id, company, tracking_no, status) |
| t_after_sale | 售后(id, order_id, type, reason, status, amount) |
| t_coupon_template | 优惠券模板(id, name, type, threshold, value, total_count) |
| t_user_coupon | 用户优惠券(id, user_id, template_id, status, expire_time) |
| t_seckill_session | 秒杀场次(id, sku_id, price, stock, start_time, end_time) |

分支一和分支二用单库 ecommerce；分支三和四用三库。
