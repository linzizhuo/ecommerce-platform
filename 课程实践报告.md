### 1.2 MV架构实现

MV（Model-View）是两层架构，使用 **SpringBoot + Thymeleaf** 实现。Controller直接调用Mapper操作数据库，无独立Service层，页面由Thymeleaf服务端渲染，前后端代码耦合在同一工程。

**请求流程：** 浏览器 → Controller → Mapper → MySQL → Thymeleaf模板渲染 → HTML响应

**项目结构：**
```
src/main/java/com/cloudmall/
├── controller/    ProductController.java / UserController.java
├── mapper/        ProductMapper / UserMapper / CategoryMapper / SkuMapper
├── entity/        Product / Sku / Category / User
└── CloudMallApplication.java

src/main/resources/
├── templates/     product/list.html, detail.html / user/login.html, register.html
└── application.yml
```

**已实现功能：** 商品列表(th:each循环渲染)、商品详情(SKU规格表格)、分类导航、关键词搜索、用户注册登录(Session保持登录态)。

**局限性：** 业务逻辑散落Controller中难复用；无事务管理；密码明文存储；前后端紧耦合——引出下一阶段MVC升级。

## 2. MVC架构设计与实现

### 2.1 MVC架构搭建

在MV基础上增加Service业务逻辑层，形成标准三层MVC架构。

| 对比维度 | MV架构 | MVC架构 |
|---------|--------|---------|
| 分层 | 2层(Controller+Mapper) | 3层(+Service) |
| Controller职责 | 含业务逻辑 | 仅参数校验+路由转发 |
| 事务 | 无 | @Transactional |
| 返回格式 | ModelAndView | 统一R\<T\> JSON |
| 异常处理 | 默认白页 | @RestControllerAdvice |
| 密码 | 明文 | BCrypt加密 |
| 认证 | Session | JWT无状态Token |

**新增公共模块：** R.java统一返回体 / BusinessException+GlobalExceptionHandler全局异常 / JwtInterceptor+WebConfig认证与跨域 / JwtUtil工具类。

### 2.2 注册功能模块实现

**流程：** Controller接收JSON → Service校验手机号唯一性+密码≥6位 → BCrypt加密 → Mapper写入 → 签发JWT(注册即登录)。

```java
public R<String> register(String phone, String password, String nickname) {
    if (userMapper.selectOne(eq(User::getPhone, phone)) != null)
        throw new BusinessException("手机号已注册");
    if (password.length() < 6) throw new BusinessException("密码长度不能少于6位");
    User user = new User();
    user.setPhone(phone);
    user.setPassword(BCrypt.hashpw(password, BCrypt.gensalt()));
    user.setNickname(nickname);
    userMapper.insert(user);
    return R.ok(JwtUtil.generate(user.getId(), user.getPhone()));
}
```

BCrypt每次生成随机盐值内置密文中，相同密码加密结果不同，有效防彩虹表攻击。

### 2.3 登录模块实现

**流程：** phone+password → 查询用户 → `BCrypt.checkpw`验证密码 → 检查status状态 → 签发JWT(24h有效)。

JWT结构：Header(HS256) + Payload(userId/phone/iat/exp) + Signature(密钥签名)，Token自包含用户信息，服务端无需Session。

**JwtInterceptor：** 从Authorization头提取`Bearer <token>` → 验证签名与有效期 → 解析userId → 验证失败返回401。公开接口(登录/注册/商品查询)通过excludePathPatterns放行。

**前端Axios拦截器：** 请求拦截器自动注入Token头；响应拦截器处理401清除Token跳转登录页。

## 3. 基于SpringBoot+Vue的网上购物系统的设计与实现

### 3.1 业务需求分析

**系统定位：** B2B2C多商户电商平台，支持用户前台购物、商家后台管理、平台运营管理。

| 角色 | 核心功能 |
|------|---------|
| 普通用户(C端) | 注册登录 → 浏览/搜索 → SKU选择 → 购物车 → 领券 → 下单支付 → 订单管理 → 评价 → 售后 |
| 商家(B端) | 商品管理(发布/SKU/上下架) → 订单发货 → 优惠券 → 数据看板 |
| 平台管理员 | 用户管理 → 商品审核 → 商家管理 → RBAC权限 → 数据大屏 |

**核心业务闭环：** 注册/登录 → 浏览商品(分类/搜索) → 商品详情 → 加入购物车 → 领券 → 结算(地址+券+支付方式) → 模拟支付 → 商家发货 → 确认收货 → 评价。

### 3.2 数据库设计

系统共设计 **20张表**，建立 **18条外键约束**，分四大业务域。金额以INT(分)存储避免精度问题，订单表快照冗余确保历史数据不受商品变更影响。全部使用InnoDB引擎+utf8mb4字符集。

### 5.2 主要功能代码实现

**1. Redis购物车（CartServiceImpl）**

使用Redis Hash结构，Key=`cart:{userId}`，Field=`skuId`，Value=商品信息JSON。选择Hash而非MySQL的理由：内存读写<1ms(磁盘~10ms)，支持10万QPS，可设TTL自动过期。

```java
public void addItem(Long userId, Long skuId, ...) {
    String key = "cart:" + userId;
    Object exist = redisTemplate.opsForHash().get(key, skuId.toString());
    if (exist != null) {
        // 已存在则数量累加，更新价格
        Map<String, Object> item = JsonUtil.toMap((String) exist);
        item.put("quantity", (Integer) item.get("quantity") + quantity);
        item.put("price", price);
        redisTemplate.opsForHash().put(key, skuId.toString(), JsonUtil.toJson(item));
    } else {
        // 新商品构建条目写入
        Map<String, Object> item = new LinkedHashMap<>();
        item.put("skuId", skuId); item.put("productName", productName);
        item.put("price", price); item.put("quantity", quantity);
        item.put("checked", true);
        redisTemplate.opsForHash().put(key, skuId.toString(), JsonUtil.toJson(item));
    }
}
```

**2. 分布式锁防超卖（Redisson+乐观锁）**

```java
RLock lock = redissonClient.getLock("stock:" + skuId);
try {
    if (lock.tryLock(3, 5, TimeUnit.SECONDS)) {
        Sku sku = skuMapper.selectById(skuId);
        if (sku == null || sku.getStock() < quantity)
            throw new BusinessException("库存不足");
        int rows = skuMapper.deductStock(skuId, quantity); // WHERE stock >= #{quantity}
        if (rows == 0) throw new BusinessException("库存扣减失败");
    }
} finally { if (lock.isHeldByCurrentThread()) lock.unlock(); }
```

三层保障：Redisson分布式锁（单线程操作）→ MySQL乐观锁（`stock>=quantity`）→ @Transactional（失败回滚）。

**3. 优惠券计算：** 满减券(type=1)：discount = value；折扣券(type=2)：discount = total × (100-value)/100（value=85=8.5折）。前端computed实时校验门槛。

**4. Axios+JWT认证：** 请求拦截器从localStorage取Token注入`Authorization: Bearer <token>`；响应拦截器code≠200弹错误提示，401清除Token跳登录。后端JwtInterceptor校验签名与有效期后解析userId。

**5. 全局异常处理：** @RestControllerAdvice拦截BusinessException返回业务错误，Exception兜底返回"服务器内部错误"，避免堆栈信息泄露。

6. **数据库设计：** 20表完整模型，SPU-SKU分离，JSON规格，价格分存储，订单快照，三库微服务规划

### 不足与展望

高级营销(拼团/秒杀/预售)未完整实现；支付/物流使用模拟数据；微服务拆分处规划阶段；搜索可升级Elasticsearch；未进行JMeter压测。
