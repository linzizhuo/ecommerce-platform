1.2 MV架构实现


MV（Model-View）架构是两层架构模式，Model 负责数据访问，View 负责页面渲染。本阶段使用 SpringBoot + Thymeleaf 技术栈实现。


MV架构特点：

  • Controller 直接调用 Mapper 进行数据库操作
  • 没有独立的 Service 业务逻辑层
  • 页面由 Thymeleaf 模板引擎在服务端渲染
  • 前端页面与后端代码耦合在同一工程中

项目核心代码结构：

2.1 MVC架构搭建


在MV架构基础上，增加 Service 业务逻辑层，形成标准的三层 MVC 架构。


MVC三层对比MV两层：


对比维度MV架构MVC架构
分层数量2层3层（+Service）
Controller职责直接调Mapper，含业务逻辑仅参数校验+路由转发
业务逻辑位置散落在Controller集中在Service层
事务管理无@Transactional
返回格式ModelAndView统一R\ JSON
异常处理默认白页@RestControllerAdvice全局处理
密码存储明文BCrypt加密
认证方式SessionJWT无状态Token

MVC架构请求流程：

浏览器 → Controller → Service → Mapper → MySQL
                ↓          ↓
         参数校验    业务逻辑+事务
                ↓
         统一R JSON返回

Controller层核心代码示例：

@RestController
@RequestMapping("/api/user")
public class UserController {
    @Resource
    private UserService userService;

    @PostMapping("/login")
    public R login(@RequestBody Map body) {
        return userService.login(
            (String) body.get("phone"),
            (String) body.get("password")
        );
    }
}

2.2 注册功能模块实现


注册流程：

  1. Controller 接收 JSON 请求体 {phone, password, nickname}
  2. Service 层进行业务校验（手机号唯一性、密码长度≥6）
  3. BCrypt 加密密码：BCrypt.hashpw(password, BCrypt.gensalt())
  4. Mapper 写入 MySQL 数据库
  5. 返回 JWT Token 实现注册即登录

核心代码（Service层）：

public R register(String phone, String password, String nickname) {
    User exist = userMapper.selectByPhone(phone);
    if (exist != null) throw new BusinessException("手机号已注册");
    if (password.length() < 6) throw new BusinessException("密码长度不能少于6位");

    User user = new User();
    user.setPhone(phone);
    user.setPassword(BCrypt.hashpw(password, BCrypt.gensalt()));
    user.setNickname(nickname);
    userMapper.insert(user);

    String token = JwtUtil.generate(user.getId(), user.getPhone());
    return R.ok(token);
}

2.3 登录模块实现


登录流程：

  1. Controller 接收 phone + password
  2. Service 根据手机号查询用户
  3. BCrypt 校验密码：BCrypt.checkpw(password, user.getPassword())
  4. 签发 JWT Token（有效期24小时）
  5. 前端存储 Token 到 localStorage，后续请求通过 Authorization Header 携带

JWT认证拦截器：

@Component
public class JwtInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, ...) {
        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {
            token = token.substring(7);
        }
        if (token == null || !JwtUtil.validate(token)) {
            response.setStatus(401);
            return false;
        }
        request.setAttribute("userId", JwtUtil.getUserId(token));
        return true;
    }
}

5.2 主要功能代码实现


1. Redis购物车实现（CartServiceImpl.java）：


购物车使用 Redis Hash 结构存储，Key 为 cart:{userId}，Field 为 skuId，Value 为商品详情 JSON。


public void addItem(Long userId, Long skuId, String productName,
                    String specInfo, Integer price, Integer quantity, String image) {
    String key = "cart:" + userId;
    Object existing = redisTemplate.opsForHash().get(key, skuId.toString());
    if (existing != null) {
        // 已存在则数量累加
        Map item = (Map) existing;
        item.put("quantity", (Integer) item.get("quantity") + quantity);
        redisTemplate.opsForHash().put(key, skuId.toString(), item);
    } else {
        // 新增商品到购物车
        Map item = new HashMap<>();
        item.put("skuId", skuId);
        item.put("productName", productName);
        item.put("price", price);
        item.put("quantity", quantity);
        item.put("checked", true);
        redisTemplate.opsForHash().put(key, skuId.toString(), item);
    }
}

2. 分布式锁防超卖（Redisson）：


下单时使用 Redisson 分布式锁 + MySQL 乐观锁双重保障：


RLock lock = redissonClient.getLock("stock:" + skuId);
try {
    if (lock.tryLock(3, 5, TimeUnit.SECONDS)) {
        Sku sku = skuMapper.selectById(skuId);
        if (sku == null || sku.getStock() < quantity) {
            throw new BusinessException("库存不足");
        }
        sku.setStock(sku.getStock() - quantity);
        skuMapper.updateById(sku);
    }
} finally {
    if (lock.isHeldByCurrentThread()) lock.unlock();
}

4. 前端优惠券门槛校验：


const discount = computed(() => {
  const c = coupons.value.find(c => c.id === selectedCouponId.value)
  if (!c || total.value < c.threshold) return 0  // 未满门槛
  if (c.type === 1) return c.value  // 满减券
  if (c.type === 2) return Math.floor(total.value * (100 - c.value) / 100)  // 折扣券
  return 0
})

  3. 后端分层设计： Controller-Service-Mapper 三层解耦，统一返回体 R\、全局异常处理
  4. 高并发方案： Redis 缓存策略、Redisson 分布式锁防超卖、优惠券门槛计算
  5. 安全认证： BCrypt 密码加密、JWT 无状态认证、拦截器统一鉴权
  6. 数据库设计： 三库设计为微服务拆分做准备，20张表涵盖电商全部业务

解决的问题：

  • SKU 规格 JSON 存储与前端格式化显示
  • 购物车 Redis Hash 结构的选型优势
  • 下单时分布式锁 + 乐观锁防止超卖
  • 满减券与折扣券的计算差异化处理
  • 前后端分离下的 CORS 代理配置

不足与展望


  1. 拼团、秒杀、预售等高级营销功能尚未实现
  2. 真正的第三方支付/物流接口使用了模拟数据
  3. 微服务拆分还停留在规划阶段
  4. 搜索引擎可升级为 Elasticsearch 替代 MySQL LIKE

  • Node.js 18+
  • MySQL 8.0

启动步骤


1. 初始化数据库：

mysql -u root -p < init.sql

2. 配置连接信息：

修改 src/main/resources/application.yml 中的 MySQL 和 Redis 连接地址。


3. 启动后端：

mvn spring-boot:run
# 后端运行在 http://localhost:8080

4. 启动前端：

cd vue-frontend
npm install
npx vite --host 0.0.0.0
# 前端运行在 http://localhost:3000
